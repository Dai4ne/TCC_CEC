**INTRODUÇÃO**<br>
A gestão eficiente de equipamentos escolares é essencial para o funcionamento adequado das atividades pedagógicas e administrativas. Na Escola Joaquim de Moura Candelária, o controle desses materiais ainda é realizado de forma manual, o que ocasiona falhas, extravios e dificuldade de rastreamento. Esse cenário evidencia a necessidade de modernização dos processos internos por meio de uma solução digital que torne o gerenciamento mais rápido, organizado e confiável.</p>
<p></p>Diante disso, este trabalho tem como objetivo desenvolver um sistema informatizado para o controle de equipamentos da Escola Candelária, permitindo registrar entradas, saídas, devoluções e manutenções de forma centralizada. A proposta justifica-se pela possibilidade de reduzir erros, ampliar a segurança da informação e facilitar a tomada de decisões através de relatórios e dados estruturados.</p><br>


**OBJETIVO**<br>
O objetivo geral deste trabalho é facilitar o cotidiano dos inspetores e demais colaboradores da instituição de ensino, por meio do desenvolvimento de um sistema web que proporciona uma forma simples, eficiente e organizada de realizar o agendamento e o controle dos empréstimos de equipamentos escolares. O sistema busca substituir o processo manual anteriormente realizado em papel, promovendo uma gestão digital mais ágil, segura e acessível. Além disso, pretende-se otimizar o tempo e reduzir falhas humanas, garantindo um registro centralizado das movimentações e oferecendo maior praticidade aos usuários. A proposta também visa atender aos princípios de acessibilidade, tornando o uso do sistema intuitivo e funcional mesmo para funcionários com menor familiaridade com tecnologias digitais.<br>

**TECNOLOGIAS USADAS**<br>
Para o desenvolvimento do sistema, foram utilizadas diversas ferramentas e linguagens de programação, cada uma com função específica no processo de construção do aplicativo. O Visual Studio Code foi utilizado como ambiente de desenvolvimento integrado, permitindo a edição e organização do código. O MySQL Workbench auxiliou na modelagem e gerenciamento do banco de dados, enquanto o XAMPP possibilitou a criação de um servidor web local para testes e execução do sistema. As linguagens JavaScript e PHP 8 foram empregadas no desenvolvimento das funcionalidades, sendo o JavaScript responsável pela interação e dinamismo das páginas e o PHP 8 pela comunicação com o banco de dados e processamento no servidor. Para a construção da interface, utilizaram-se o HTML5, CSS3 e Bootstrap, garantindo estrutura, estilo e design responsivo. O Git e o GitHub foram aplicados no controle de versão e na colaboração entre os desenvolvedores, assegurando a integração e o gerenciamento eficiente do código.<br>
